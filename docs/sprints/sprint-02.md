## 7. Evolucao da aplicacao web
Nesta sprint, a aplicação web deixou de ser apenas um documento e passou a ser algo materializado estrutural e visualmente. A evolução principal ocorreu na definição da interface do usuário com os protótipos e no diagrama do banco de daods (MER/DER). Além disso, o software teve seu primeiro incremento real de código com a construção da seção inicial (Hero) no Front-end, estabelecendo a base do projeto para a codificação das próximas telas.
## 8. Dificuldades encontradas

As principais dificuldades encontradas em cada área:
Design: A parte da grade ficou bem complexa de fazer um protótipo no tempo da sprine
Dados: As relações das tabelas foi um problema e algumas dúvidas surgiram no padrão de nomenclatura.
Back-end: Dificuldades em mapear quais permissões serão feitas no back-end e quais ficariam para o front-end.

## 9. Pendencias para a proxima sprint
Algumas coisas ficaram pendentes para finalizar na próxima sprint, como: Finalizar o protótipo da tela de montagem das grades, continuar os estudos de scraping com python e finalizar o modelo relatcional do banco de dados.
